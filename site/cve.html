Description

A command injection vulnerability in the FortiManager fgfm daemon allows an unauthenticated remote attacker to execute arbitrary OS commands as root by sending a crafted request to the management interface listening on TCP/541. Successful exploitation provides full control of the FortiManager appliance, which in turn manages configuration across all administered FortiGate devices.

The flaw exists in the request-parsing routine, which interpolates a tenant-supplied field into a shell command without sanitisation. Exploitation does not require valid credentials. The vendor advisory rates the issue 9.8 / Critical and recommends immediate patching; affected versions span FortiManager 7.0 through 7.6, and FortiManager Cloud. Mandiant attributes pre-disclosure exploitation to UNC5820, a suspected China-nexus operator.

Mitigations

  • Patch. Upgrade to the fixed version corresponding to your major release.
  • Restrict the management interface. Block TCP/541 at the perimeter from any source that does not require fgfm.
  • Rotate credentials. Assume any administered FortiGate credentials reachable from FortiManager are compromised; rotate API tokens and admin passwords.
  • Hunt. Search for the diag_npu child-process patterns documented in the Mandiant write-up.
